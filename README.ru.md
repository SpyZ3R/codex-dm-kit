# codex-dm-kit

[English](README.md)

`codex-dm-kit` — неофициальный набор для одиночных локальных кампаний в Codex: DnD 5e light, честные броски, файловая память, строгий учёт знаний NPC и HTML-карточка, работающая прямо с диска.

Git используется только для получения исходного kit. Кампания остаётся в выбранной пользователем локальной папке и не получает `.git`. Сам kit не загружает файлы кампании.

Требования: Codex с локальной папкой проекта, Git и Python 3.10 или новее.

## Быстрый старт

1. Создайте или выберите локальную папку будущей кампании.
2. Откройте именно эту папку как проект в Codex.
3. Скопируйте следующий блок первым сообщением.

```text
Создай в текущей открытой папке локальную одиночную кампанию с помощью codex-dm-kit.

Источник:
https://github.com/SpyZ3R/codex-dm-kit

Релиз:
v0.1.1

Действуй самостоятельно:

1. Определи абсолютный путь корня текущего проекта и покажи его мне.
2. Используй этот корень как папку кампании. Не предлагай другой каталог без необходимости.
3. Проверь, что выбранная папка не находится внутри Git-worktree и сама не содержит .git. Кампания не должна использовать Git.
4. Осмотри уже существующие файлы. Не удаляй и не перезаписывай их без моего подтверждения. Подходящие описания персонажа и сеттинга используй как материалы для импорта.
5. Если в папке уже существует инициализированная кампания codex-dm-kit, не запускай Session Zero повторно — загрузи её и продолжай.
6. Проверь наличие Git и Python 3.10+.
7. Временно клонируй точный релиз codex-dm-kit с помощью shallow clone в системный временный каталог и проверь, что HEAD соответствует указанному тегу. Не клонируй его внутрь моей папки проекта.
8. Полностью прочитай AGENTS.md и README.ru.md из временной копии.
9. Проведи разговорную Session Zero, задавая не более трёх коротких вопросов за один ответ.
10. Предложи создать персонажа и сеттинг вместе либо импортировать мои материалы.
11. Уточни язык, тон, границы, опасность и предпочтения по исследованию, диалогам, боям и тайнам.
12. Перед созданием файлов покажи итоговую концепцию и получи моё подтверждение.
13. После подтверждения передай нормализованный JSON Session Zero генератору через stdin и создай файлы кампании непосредственно в корне открытого проекта. Не сохраняй ответы внутри clone или в другом временном файле.
14. Не выполняй git init, не загружай файлы кампании в сеть и не создавай внешних копий.
15. Запусти валидацию и сформируй dashboard.
16. Прочитай созданные AGENTS.md и main_prompt.md и сразу начни первую игровую сцену в этом же чате.
17. После успешной инициализации удали только созданную тобой временную копию codex-dm-kit.
```

## Что спросит Codex

Session Zero позволяет создать персонажа и мир с нуля или импортировать готовые материалы. Codex уточнит язык, тон, границы, уровень опасности, желаемое соотношение исследования, общения, боя и тайн, затем покажет итоговую концепцию до создания файлов.

## Что создаётся в выбранной папке

- `AGENTS.md` и `main_prompt.md` — постоянный контракт мастера;
- `player_state.json`, `npcs.json`, `quests.json`, `world_state.json`, `locations.json`, `battle_state.json` — актуальное состояние;
- `session_log.md` — неизменяемая история установленных событий;
- `campaign_summary.md`, `rules_notes.md`, `secrets.md`, `image_prompts.md`;
- `tools/codex_dm.py` — броски, проверка и сборка карточки;
- `dashboard/index.html` и генерируемый `dashboard_data.js`.

Полный обезличенный пример входных данных находится в [`examples/onboarding_answers.ru.json`](examples/onboarding_answers.ru.json).

После значимого игрового события Codex синхронизирует затронутые файлы, дописывает журнал и запускает `finalize`. Внеигровой вопрос не должен менять состояние.

## Ручные команды

Из временной или постоянной копии kit:

```powershell
python -m codex_dm_kit inspect-target --target D:\путь\к\выбранному\проекту
python -m codex_dm_kit create --answers answers.json --target D:\путь\к\выбранному\проекту
```

Автоматический bootstrap использует `--answers -` и передаёт JSON через stdin, поэтому данные Session Zero не копируются во временный checkout kit.

Внутри созданной кампании:

```powershell
python tools/codex_dm.py roll 1d20+5
python tools/codex_dm.py validate
python tools/codex_dm.py finalize
```

На Windows можно использовать `py` вместо `python`.

## Как продолжить позже

Снова откройте ту же папку кампании как проект Codex и попросите продолжить игру. Созданный `AGENTS.md` загрузит существующее состояние и не позволит повторно запустить Session Zero. После `finalize` обновите уже открытую вкладку dashboard, чтобы увидеть новое состояние.

## Приватность и безопасность

- Генератор отказывается создавать кампанию внутри Git-worktree.
- Совпадающие системные файлы не перезаписываются.
- Остальные пользовательские файлы сохраняются без изменений.
- Dashboard строится по белому списку и не включает `secrets.md`, скрытые квесты и неизвестные персонажу сведения.
- Kit не требует OpenAI API-ключа: он использует уже открытый Codex.
- Если кампания должна оставаться только на одном устройстве, выберите действительно локальную несинхронизируемую папку. OneDrive, Dropbox, сетевые диски, резервное копирование и сам используемый продукт могут обрабатывать данные по собственным правилам.
- `secrets.md` — обычный текстовый файл для разделения спойлеров мастера, а не шифрование или контроль доступа. Не храните в кампании API-ключи, пароли, приватные ключи и другие реальные учётные данные.
